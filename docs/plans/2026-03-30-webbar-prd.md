# WebBar 产品需求文档（PRD）

- 文档版本：v1.0（草案）
- 日期：2026-03-30
- 产品名称：WebBar
- 目标平台：macOS

## 1. 背景与目标

WebBar 是一个 macOS 状态栏效率工具。核心价值是把“打开常用网页”从“多步骤操作（找浏览器/找书签/点菜单）”压缩为“单击状态栏图标”。

本产品的设计原则是：

1. 单击直达，不允许先点图标再进下拉菜单选网址。
2. 多网址场景通过多个状态栏图标并列展示，而不是菜单列表。
3. 配置能力集中在独立管理页，日常使用路径尽可能短。

## 2. 目标用户

1. 高频访问固定网页的人群（运营、客服、开发、设计、交易等）。
2. 有多站点快速切换需求，但不希望依赖浏览器书签栏的人群。
3. macOS 桌面效率工具重度用户。

## 3. 产品范围

### 3.1 MVP（本期必须）

1. 状态栏显示最多 5 个网址图标。
2. 左键单击任意图标，使用系统默认行为打开该 URL（调用系统默认浏览器处理）。
3. 右键任意图标，进入“管理网址”页面。
4. 管理页支持新增、编辑、删除网址。
5. 图标自动抓取网站 favicon；抓取失败时允许用户设置 emoji 作为图标。
6. 支持任意 URL scheme（不限于 `http/https`）。
7. 打开失败时：图标短暂变红 + 系统通知提示错误。
8. 数据仅本地存储，不做账号体系与云同步。
9. 开机启动默认关闭，但可在设置中开启。

### 3.2 非目标（本期不做）

1. 状态栏下拉菜单选网址。
2. iCloud 或自建账号同步。
3. 智能切换已打开标签页。
4. 超过 5 个图标的状态栏展示策略。

## 4. 核心用户故事

1. 作为用户，我希望单击状态栏图标就立即打开目标网页，不经过二级菜单。
2. 作为用户，我希望可在应用内管理常用网址，并让这些网址以多个图标出现在状态栏。
3. 作为用户，我希望图标尽量是站点 favicon，失败时可改用 emoji，保证可识别性。
4. 作为用户，我希望打开失败时有明确反馈，知道哪里出错并可修复。

## 5. 功能需求

### 5.1 状态栏交互

1. 每个网址对应一个独立状态栏图标（`NSStatusItem`）。
2. 左键点击图标：立即打开绑定 URL（系统默认浏览器处理）。
3. 右键点击图标：显示快捷操作，至少包含“管理网址”“退出 WebBar”。
4. 状态栏图标数量上限为 5；超过上限时在管理页阻止保存并提示原因。

### 5.2 管理网址页面

1. 字段：名称（可选）、URL（必填）、图标模式（自动 favicon / emoji）。
2. 新增时校验 URL scheme 合法性（允许任意 scheme，但需可被系统 `open` 调用）。
3. 编辑时支持切换图标模式。
4. 删除后状态栏实时移除对应图标。
5. 列表顺序即状态栏图标顺序（默认按创建顺序，可在后续版本增加排序能力）。

### 5.3 图标策略

1. 优先使用 favicon（缓存到本地）。
2. favicon 拉取失败或解析失败时，使用默认占位图标并提示可设置 emoji。
3. 用户设置 emoji 后，优先使用 emoji，不再自动覆盖。

### 5.4 启动与设置

1. 提供“开机启动”开关，默认关闭。
2. 设置项本地持久化，重启后生效。

### 5.5 异常与反馈

1. URL 无法打开时，图标短暂变红（例如 1 秒）。
2. 同时触发系统通知，文案包含失败网址与建议操作（检查 URL 或默认浏览器设置）。
3. 管理页对格式错误/空值给出即时校验提示。

## 6. 数据与存储

### 6.1 本地数据模型（建议）

- `LinkItem`
  - `id: UUID`
  - `name: String?`
  - `url: String`
  - `iconType: favicon | emoji`
  - `emoji: String?`
  - `faviconCachePath: String?`
  - `createdAt: Date`
  - `updatedAt: Date`

### 6.2 存储策略

1. 使用本地持久化（如 `UserDefaults` + 文件缓存，或 `SQLite/CoreData`）。
2. 不上传云端，不跨设备同步。
3. favicon 缓存可按 URL 域名维度更新，避免频繁拉取。

## 7. 非功能需求

1. 性能：应用冷启动后 1 秒内完成状态栏图标渲染（≤5 个）。
2. 稳定性：单个网址打开失败不影响其他图标可用。
3. 可用性：主要操作路径（打开网页）保持单击完成。
4. 隐私：仅本地存储网址数据，不采集浏览行为。

## 8. 验收标准（MVP）

1. 添加 1 个网址后，状态栏出现 1 个图标，左键可成功打开网页。
2. 添加 5 个网址后，状态栏出现 5 个图标；第 6 个网址无法保存并有清晰提示。
3. 右键任意图标可进入管理页并完成增删改，状态栏实时同步。
4. favicon 拉取失败时可手动设置 emoji，设置后状态栏正常显示 emoji。
5. 输入 `mailto:`、`obsidian:` 等非 `http/https` URL，可成功调用系统处理器打开。
6. 打开失败时，出现图标变红反馈与系统通知。
7. 开机启动默认关闭；用户开启后重启系统可生效。

## 9. 里程碑建议

1. M1（原型）：
   - 单网址图标 + 左键打开 + 本地配置
2. M2（MVP）：
   - 多图标（上限 5）+ 管理页 + favicon/emoji + 错误反馈
3. M3（优化）：
   - 图标缓存优化 + 管理页体验打磨 + 启动性能优化

## 10. 后续可选演进（非本期）

1. 拖拽排序与分组。
2. 导入导出配置。
3. 快捷键打开最近网址。
4. iCloud 同步。
